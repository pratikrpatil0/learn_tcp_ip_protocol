\section{Key Findings}

Five clear takeaways come from the visuals and tables:

\begin{enumerate}
    \item Acknowledgements and timeouts make the system much more reliable.
    \item Chunked sending and sliding windows are what push throughput up.
    \item Receiver buffering in v11+ reduces waste when packets arrive out of
    order.
    \item Adaptive RTO cuts down unnecessary retransmissions when delay
    changes.
    \item Fast retransmit/recovery, first added in v14 and refined through
    v24, reacts to loss faster than timeout-only recovery.
\end{enumerate}

\section{Limitations}

\begin{enumerate}
    \item Most tests ran on localhost, so they do not show full WAN variety.
    \item Security features such as authentication and encryption are left out
    on purpose.
    \item A full TCP-style advertised window and SACK block encoding are out
    of scope here.
    \item The charts are useful for analysis, but a fully automated benchmark
    harness is still future work.
\end{enumerate}

\section{Conclusion}

This chapter shows that the project goes from simple datagram exchange to a
strong educational transport model that is reliable, can handle multiple
requests at once, can adapt, and is aware of congestion. The final versions are
useful for both putting into practice and learning about transport protocols.

One of the most important final results is the interactive visualizer. It brings
together the sender code, receiver code, finite state machine, and UML timeline
into one clear educational view. That makes the protocol easier for new
students to learn, since TCP, congestion control, and C-based socket
programming can be hard to understand when you only read about them or look at
code. In this way, the project is more than just a piece of software; it's also
a useful teaching tool that makes the ideas clear, organized, and easier to
remember.

As the last step in the project, I made both the protocol code and the
interactive visualizer for use in the classroom. Students find it easier to
understand networking concepts when they have versioned C implementations,
textbook-based references, and synchronized visual explanations. This is
especially helpful for beginners because the visual walkthrough makes C and
transport-protocol ideas easier to understand and turns the system into a
step-by-step learning experience.
